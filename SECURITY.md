# Security Configuration Guide

This project contains sensitive information that should never be committed to version control.

## Protected Files

The following files contain sensitive API keys and should **NEVER** be committed:

1. **google-services.json** (both app-child and app-parent)
   - Contains Firebase configuration and API keys
   - Download from Firebase Console for each app

2. **local.properties**
   - Contains SDK path and API keys
   - Generated locally with your specific configuration

3. **supabase/config.toml**
   - Contains Supabase configuration
   - Generated by Supabase CLI

## Setup Instructions

### 1. Firebase Configuration

1. Go to [Firebase Console](https://console.firebase.google.com/)
2. Select your project
3. Download `google-services.json` for each app:
   - For Child App: Place in `app-child/`
   - For Parent App: Place in `app-parent/`
4. Use the `.example` files as templates

### 2. Local Properties

1. Copy `local.properties.example` to `local.properties`
2. Update with your actual values:
   ```properties
   sdk.dir=YOUR_SDK_PATH
   MAPS_API_KEY=YOUR_ACTUAL_MAPS_KEY
   SUPABASE_URL=YOUR_SUPABASE_URL
   SUPABASE_ANON_KEY=YOUR_SUPABASE_ANON_KEY
   ```

### 3. Google Maps API Key

1. Get your API key from [Google Cloud Console](https://console.cloud.google.com/)
2. Enable Maps SDK for Android
3. Add to `local.properties`:
   ```properties
   MAPS_API_KEY=YOUR_KEY_HERE
   ```

### 4. Supabase Configuration

1. Get your credentials from [Supabase Dashboard](https://app.supabase.com/)
2. Add to `local.properties`:
   ```properties
   SUPABASE_URL=https://your-project.supabase.co
   SUPABASE_ANON_KEY=your-anon-key
   ```

## Git Security

The `.gitignore` file is configured to exclude:
- `google-services.json` files
- `local.properties`
- `supabase/config.toml`
- All `.env` files
- Keystore files

## Already Committed Sensitive Data?

If you've already committed sensitive data:

1. Remove from git history:
   ```bash
   git filter-branch --force --index-filter \
     "git rm --cached --ignore-unmatch app-child/google-services.json app-parent/google-services.json" \
     --prune-empty --tag-name-filter cat -- --all
   ```

2. Force push (WARNING: This rewrites history):
   ```bash
   git push origin --force --all
   ```

3. Regenerate all exposed API keys immediately from their respective consoles

## Best Practices

- Never commit API keys or secrets
- Use environment variables for CI/CD
- Rotate keys if accidentally exposed
- Use different keys for development/production
- Review commits before pushing
- Enable secret scanning on GitHub

## Need Help?

If you accidentally committed secrets:
1. Rotate the keys immediately
2. Remove from git history
3. Update all team members
